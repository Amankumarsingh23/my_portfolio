Overview

🗂️
What it is
A custom VCS built from scratch that mirrors Git's core — staging, commits, branches, merges, diffs
⚙️
Language
Pure C++ — no external libraries. Raw file I/O, SHA hashing, custom data structures throughout
🎯
Goal
Deeply understand how Git works under the hood — object model, DAG commits, tree objects
// Motivation

Most developers use Git every day without understanding what happens under the hood. MiniGit was built to answer that question from first principles — by reimplementing the core of Git in C++. The project covers everything from the staging area and SHA-based object store to branching, merging, and conflict detection, all without any external libraries.

Architecture

Directory Structure
your-repo/
  .minigit/
    objects/  ← blob/tree/commit objects
    refs/    ← branch pointers
    HEAD     ← current branch ref
    INDEX    ← staging area
    COMMIT_LOG ← commit history
Object Model
blob — stores raw file content, SHA-hashed for deduplication
tree — maps filenames to blob SHA hashes, represents a directory snapshot
commit — points to a tree + parent commit, stores metadata and message
Sample CLI Usage
$ ./minigit init           # initialise .minigit/ directory
$ ./minigit add main.cpp     # stage a file
$ ./minigit commit -m "first commit"
$ ./minigit branch feature-x  # create a branch
$ ./minigit checkout feature-x
$ ./minigit merge main       # merge with conflict detection
$ ./minigit log           # view commit history
$ ./minigit diff          # diff working tree vs HEAD

Key Features

📦
Staging Area (INDEX)
Files tracked in a persistent INDEX file — mirrors Git's staging model exactly with add/reset support
🔐
SHA-Based Object Store
Every file, tree, and commit is hashed — deduplication and integrity checking guaranteed across history
🌿
Branching & Checkout
Create and switch branches, with HEAD tracking the current branch pointer — supports full parallel workflows
🔀
Merge & Conflict Detection
Merges branches and flags conflicts when the same file has divergent changes on both sides
📜
Commit Log
Persistent commit history stored as a DAG — traverse log, view messages, timestamps, and parent hashes
🔍
Diff Viewer
Line-by-line diff between the working tree and HEAD — shows added and removed lines clearly
