How I Approach Problems

01
Understand constraints first
Array size, value range, time limit — these dictate the algorithm before writing a single line
02
Brute force → optimise
Start with the naive O(n²) or O(n³) solution, then identify the bottleneck and improve it
03
Edge cases explicitly
Empty input, single element, all duplicates, negative numbers — documented in every solution
04
Complexity analysis always
Every solution documents Big-O time and space — the habit that matters in real interviews
