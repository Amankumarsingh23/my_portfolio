Overview

📦
What it does
Compresses any file losslessly using Huffman Coding — up to 60% size reduction, fully reversible
⚙️
Core Algorithm
Greedy min-heap builds the optimal Huffman tree — frequent bytes get shorter codes, rare ones longer
⚡
Innovation
Custom bit-level I/O streams — writes individual bits, not bytes, for maximum compression density
// How it works

HuffCompress reads the input file and builds a frequency table of every byte. A min-heap then greedily constructs the optimal prefix-free Huffman tree — bytes that appear often get short codes, rare bytes get long ones. The tree is serialised into the output file header so decompression can perfectly reconstruct it. The payload is then written at the bit level, not the byte level, squeezing out maximum compression.

Algorithm Deep-Dive

Compression Pipeline
01Scan input → build byte frequency table
02Insert all bytes into a min-heap (priority queue)
03Greedily merge two lowest-freq nodes → build tree
04Traverse tree → generate prefix-free code table
05Serialise tree into file header (for decompression)
06Encode input file bit-by-bit → write compressed output
Example — "aabbbcccc"
Frequencies:
  c → 4   b → 3   a → 2
Huffman codes:
  c → 0   (1 bit — most frequent)
  b → 10  (2 bits)
  a → 11  (2 bits — least frequent)
Original: 9 bytes = 72 bits
Compressed: 14 bits → ~80% reduction
CLI Usage
$ ./huffcompress compress input.txt output.huff
  → Compressed: 84KB → 31KB (63% reduction)
$ ./huffcompress decompress output.huff restored.txt
  → Restored perfectly — lossless guaranteed
